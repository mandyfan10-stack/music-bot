## 2024-04-18 - Redundant Average Rating Calculations in Render Loops
**Learning:** The application computes the average rating of releases using O(N) `reduce` operations inside the `renderReleaseCard` function and `getAvgRating` (used heavily in sorting algorithms like `rating-desc` and `rating-asc`). This causes redundant O(N * M) calculations where N is the number of releases and M is the number of reviews when rendering the UI or sorting, creating a performance bottleneck for large datasets.
**Action:** Precalculate and memoize the average ratings in a Map (`avgRatingByRelId`) during the `updateReviewsMap()` call (which runs only when reviews data changes). Update functions to use O(1) map lookups instead of computing averages on the fly during render or sort cycles.
